- JDK，JRE，JVM三者之间的关系，以及JDK，JRE包含的主要结构有哪些

> 它们整体上是包含的关系，JDK包含JRE,而JRE包含JVM；

> JDK=JRE+Java开发工具集（javac.exe，java.exe，javadoc.exe）；

> JRE=JVM+Java核心类库

- 为什么要配置path环境变量？如何配置？

> 为了使得有关java的命令(包括javac,java,javadoc等)可以在任何文件目录下执行，配置path环境变量后如果首先在当前文件夹下找不到可执行程序，则会环境变量path下添加的文件夹中去寻找，则可实现任何文件夹下都可执行java相关的命令行语句

> JAVA_HOME=bin的上一层目录

> path=%JAVA_HOME%\bin

- 常用的几个命令行操作都有哪些？（至少4个）

> dir：列出当前目录下的文件及文件夹

> cd：进入指定目录

> md：创建目录

> rd：删除目录

> del：删除文件

> cd .. 退回到上一级目录

> cd / 退回到根目录

> exit：推出dos命令行

> echo javase>1.doc   //就是将字符串”javase“写入到当前文件夹下的1.doc文件中

- 创建如下的类，使得运行的话可以输出：

```text
姓名：习大大

姓名：男
家庭住址：北京中南海
```

> 创建一个java文件：ChairMan.java，代码如下：

```java
public class ChairMan{
	public static void main(String[] args){
		System.out.println("姓名：习大大\n\n姓名：男\n家庭住址：北京中南海");
	}
}
```

- 编译和运行上述代码的指令

```bash
javac ChairMan.java
java ChairMan
```

- 编译可能出错的情况：自己编写源代码是在某一IDE中写的，指定了包名，不指定的话，IDE会提示代码错误，然后我们就写了包名，然后通过命令行语句去执行，就会出现错误

- 运行可能会出现错误：找不到主类，那就是该文件名和该文件里的public类，名字不一致。另外，在输出的时候也可能乱码，那就是源代码的文件保存的格式有问题，中文的话，源代码要保存为ANSI格式，当然也可以保存为utf-8，但是在编译的时候要指定编译的格式，才不会出问题，编译命令如下：

```text
javac -encoding "utf-8" ChairMan.java
```

- javac -d 文件夹路径 源代码名.java，这个命令可以指定编译后的文件放在哪里，javac -d . ChairMan.java表示放在与ChairMan.java相同的父目录下，当然这也不是绝对的正确说法，仅仅只是在没有指定包名的情况下，如果有包名，则会在这个源代码文件夹下建立包名文件夹，然后把class文件放在包名文件夹下

```text
//如果包名a.b,，两组命令对应为
javac -d . ChairMan.java
java a.b.ChairMan

//如果包名为a.b,想把class放在源代码文件下相同位置，则
javac -d ../../ ChairMan.java
java -cp ../../ a.b.ChairMan
```

- 有些IDE工具有自己设定的源代码文件夹结构和预设的编译后class文件放置的文件夹地址，如果将这些源代码文件拿来在命令行执行，则会有报错的可能性；另外，IDE一般是使用utf-8来保存源文件，所以在控制台输出的时候中文会乱码

- 在有包名a.b的情况下，直接执行javac ChairMan.java也会在当前源代码的相同父文件夹下创建class文件，但是执行java ChairMan会报错，执行java a.b.ChairMan也报错，必须执行java -cp ../../ a.b.ChairMan才可成功。
